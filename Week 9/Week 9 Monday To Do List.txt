LF (Line Feed or \n) and CR (Carriage Return or \r).

Unix uses \n but Windows uses \r\n. Use \n since Ed is based on Linux.

Windows usually need both \n\r.

\n is a newline only, \r\n is a newline and a carriage return (i.e., move the cursor to the left).

\n is the default in Unix \r\n is the default in Windows.

I may have generated the BigestMaze.tex using Spyder in my Windows machine. That is why the \r\n.

If you generate the file using Ed, you should get \n only. Please do use \n.
